# 架构设计规约：v2.0 全域机动期

**进入 v2.0 的前置条件**

> v2.0 是 v1.0 protected TD3 / BC-assisted TD3 稳定后的下一阶段目标，不是当前 debug 的直接下一步。v2.0 暂不启动。

v2.0 必须满足以下前置���件后才能进入：

1. **v1.0 residual actor 在 gimbal_only 下稳定**：residual actor `a = clip(π_BC(s) + α Δπ(s), -1, 1)` 在 gimbal_only 模式下已通过验收，不劣于 BC baseline。
2. **critic calibration gate 通过**：critic 能正确区分好/坏动作，Q 面不出现过平 / margin 不足。
3. **recovery data / replay prefill / critic warmup 有效**：critic 在偏离状态下仍能给出合理价值估计。
4. **rollout-based model selection 已替代 best_episode_reward**：模型选择不再依赖 episode reward，而是基于固定角度 rollout eval。
5. **以下保护机制至少完成验证**：
   - actor freeze / actor update gate
   - critic warmup
   - replay prefill
   - BC regularization 或同等保护机制
   - critic calibration gate
   - residual actor 基本框架
6. **full_8d 扩展不得建立在普通 TD3 直接长训练之上**：全域机动阶段的策略复杂度远高于 v1.0，普通 TD3 的不稳定性会被放大。

**风险说明**：±180° 全域机动涉及长 horizon、奇异面穿越、飞轮加减速联动，训练难度显著高于 v1.0。如果 v1.0 阶段的 protected TD3 机制不够鲁棒，v2.0 很可能无法收敛。

---

**1. 核心目标**

放开初始条件，逼迫网络在长距离机动中遭遇 SGCMG 奇异面。通过训练，使其学会主动调动飞轮加减速（真正的 VSCMG 模式），利用动量交换强行越过奇异构型。

---

**2. 网络结构**

v2.0 不再从普通 full_8d TD3 开始。v2.0 的网络形式应为：

**初始形式（gimbal BC baseline）**：
```
a_8d = [π_BC_gimbal(s), 0_4d] + α Δπ_8d(s)
```

**full_8d BC/oracle 可用后**：
```
a_8d = π_BC_8d(s) + α Δπ_8d(s)
```

其中：
- π_BC_gimbal(s)：v1.0 得到的 gimbal_only BC baseline，前 4 维输出有效，后 4 维为 0；
- π_BC_8d(s)：如果有 full_8d 的 BC/oracle 数据，则使用 8D BC baseline；
- Δπ_8d(s)：residual actor 输出 8D residual；
  - 前 4 维：gimbal residual；
  - 后 4 维：wheel acceleration residual；
- α：residual_scale，必须保守。

---

**3. 环境初始化 (env.reset)**

* **姿态误差**：初始四元数/欧拉角在全包络域内随机生成（±180°）。

* **角速度**：赋予随机的初始翻滚角速度，增加 Detumbling 难度。



**4. 动力学与动作流转 (env.step)**

* **状态语义继承**：v2.0 完全继承 v1.0 的 22 维状态语义定义，不做任何修改。

* **双态空间解耦 (Dual-State Mapping)**：底层运动学积分强制使用四元数 (Quaternion)，利用其全域无奇异的特性保证计算鲁棒性；积分完成后，按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再通过公式 `σ_err = q_err_vec / (1 + q_err0)` 将其转换为 3 维修正罗德里格参数 (MRP)，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`），将 `σ_err` 作为姿态观测值填入 22 维 State 向量中。

* **边界感知探索噪声**：当检测到飞轮逼近物理极限（5000 rpm）时，强行截断加剧饱和方向的随机探索噪声，防止网络早期"自杀"。

* **动作平滑惩罚**：在 Reward 中激活一阶差分惩罚（基于 t 与 t-1 时刻动作的欧氏距离），严厉打击框架高频打舵。

---

**5. Critic ensemble / uncertainty gate（v2.0 强推荐或默认机制）**

* **full_8d 动作空间更大，OOD 风险更强**：从 4D 扩展到 8D，critic 面临的价值估计难度显著增加。
* **尤其监控 wheel_accel 维度的 Q uncertainty**：wheel acceleration 的 Q 值不确定性更高，容易导致 actor 在飞轮维度做出激进决策。
* **actor update 应受 conservative Q / uncertainty gate 约束**：
  - 当 critic ensemble disagreement 超过阈值时，暂停 actor 更新；
  - 使用 pessimistic Q estimate（取 ensemble 中的低 Q 值）指导 actor 更新；
  - 防止 actor 利用某个 critic 的过高估计。

---

**6. Wheel / Gimbal safety（v2.0 新增）**

* **gimbal action safety**：框架角速度不超过伺服电机物理极限；
* **wheel speed / wheel acceleration safety**：
  - 飞轮转速不允许长时间维持在物理极限附近；
  - wheel acceleration 不允许超出电机驱动能力；
* **momentum health**：
  - 不允许飞轮乱拉高转速却没有姿态收益；
  - 当系统动量管理效率低于阈值时，触发 momentum health warning / recovery-oriented objective；
  - 机动完成后飞轮转速应自动回落至基准附近。

---

**7. 验收标准**

* 卫星能平顺完成 180° 大机动。

* 机动轨迹中必须观测到：遭遇奇异时飞轮转速短促拉升（借力），到位后自动回落至基准附近。

* full_8d residual actor 不应破坏 v1.0 gimbal_only baseline 已验证的基础姿态控制能力，并应在全域机动任务中证明 wheel 维度带来有效收益。

* critic calibration gate 通过：Q 面在 8D 动作空间下仍能有效区分好/坏动作。

* wheel safety 指标不频繁触发警告。

* rollout-based model selection 验证通过。
