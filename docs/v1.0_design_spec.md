# 架构设计规约：v1.0 温室学步期

**当前修订说明：v0.5.18-debug-bc-init 后的 v1.0 路线调整**

> debug tag: `v0.5.18-debug-bc-init`

v1.0 仍是基础控制阶段，但不再假设普通 TD3 可以直接训练成功。当前已验证：

1. **BC actor 有效**：在 `env.reset(options={"init_attitude_deg": angle})` 下，BC actor 姿态恢复精度为 10°→~0.29°, 20°→~1.12°, 30°→~2.18°, 45°→~4.30°。
2. **普通 TD3 fine-tune 会破坏 BC policy**：checkpoint_10k 已严重退化，动作从小幅精确控制变成高饱和大动作。
3. **v1.0 下一阶段路线调整**：
   - 先使用 gimbal_only（4 维）模式完成基础姿态控制；
   - 采用 protected TD3 / BC-assisted TD3 训练机制；
   - 验证训练稳定后，再逐步恢复或扩展到 full_8d（8 维）。

---

**1. 核心目标**

在绝对安全、无外部干扰的物理仿真环境中，先在 gimbal_only 4 维模式下建立稳定的基础姿态控制闭环，验证 protected TD3 / BC-assisted TD3 训练机制稳定后，再逐步恢复或扩展到 full_8d 的 8 维 VSCMG 联合控制。

**2. 环境初始化 (env.reset)**

* **定位**：温室学步期 — 仍保持静止初始状态、零扰动、飞轮锁定 3000 rpm 偏置。

* **姿态误差**：采用课程学习，逐步扩大初始偏差：
  - 5° — 基础验证
  - 10° — 扩展验证
  - 20° — 扩展验证
  - 30° — 扩展验证
  - 45° — 扩展测试上限

* **角速度**：三轴本体系角速度强制归零 (0 rad/s)。

* **飞轮状态**：4 个飞轮初始转速严格锁定在 3000 rpm 标称偏置，无随机化。

* **空间扰动**：强制关闭所有重力梯度、气动阻力、光压等外部力矩。

* **评估 horizon 说明**：
  - `max_steps=320` 与 `max_steps=1000` 的评估意义不同；
  - BC actor 在 `max_steps=1000` 下可完整收敛；
  - `max_steps=320` 更多用于对齐训练 horizon 和短期改善观察。

**3. 观测状态定义（22维冻结接口）**

从 v1.0 开始，22 维状态的物理语义彻底冻结：

**22 维状态向量组成：**

- **姿态误差（3维）**：`sigma_err` — MRP 姿态误差：当前姿态相对目标姿态的误差表示
- **角速度（3维）**：`omega_B` — 本体三轴角速度 [rad/s]
- **框架编码（8维）**：`sin(delta)`（4维）+ `cos(delta)`（4维）— 4个框架角的正弦/余弦编码（消除±π断点）
- **执行器动态（8维）**：
  - `delta_dot`（4维）— 4个框架角速度 [rad/s]（为伺服延迟预留接口）
  - `Omega_w_tilde`（4维）— 4个飞轮转速相对3000rpm的归一化偏差

**总计：3 + 3 + 4 + 4 + 4 + 4 = 22维**

**各分量详细定义：**

- **`sigma_err`（3维）— MRP 姿态误差**：当前姿态相对目标姿态的误差 MRPs，通过误差四元数转换得到。按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再转换为 `σ_err = q_err_vec / (1 + q_err0)`，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`）。v1.0 阶段目标姿态固定为单位四元数，`σ_err` 即为当前姿态误差。

- **`omega_B`（3维）— 本体角速度**：航天器本体系三轴角速度，单位 rad/s。应做归一化处理，具体缩放常数由实现/配置层统一管理，跨版本保持一致。

- **`sin(delta)`（4维）、`cos(delta)`（4维）— 框架角周期编码**：4个框架角的三角函数编码。使用 `[sin, cos]` 而不是直接用 `delta` 是为了消除角度在 ±π 处的断点，保证 v2.0 全域机动时的观测连续性。

- **`delta_dot`（4维）— 框架角速度**：4个框架角的变化率，单位 rad/s。保留该项是为了让状态更接近 Markov，并为 v3.0 的伺服延迟与执行器动态建模预留接口。

- **`Omega_w_tilde`（4维）— 飞轮转速归一化偏差**：4个飞轮转速相对 3000 rpm 标称偏置的归一化偏差，`Omega_w_tilde = (Omega_w - Omega_bias) / Omega_scale`。使用"相对偏置"的形式是为了让网络更容易学习"维持偏置"的控制目标，归一化缩放因子由实现/配置层统一管理。

**冻结承诺：** v1.0~v5.0 任何版本都不改变这 22 个维量的物理含义。


**4. v1.0 子阶段划分**

v1.0 分为三个子阶段，逐步推进：

---

### v1.0-a：BC baseline 固化

* **网络形式**：`a = π_BC(s)`

* **目标**：确认 BC actor 是安全 baseline；不追求 TD3 超过 BC。

* **建立固定角度 rollout eval 标准**：
  - 使用 `env.reset(options={"init_attitude_deg": angle})` 进行固定角度 rollout 评估；
  - 记录 BC actor 在 10° / 20° / 30° / 45° 下的姿态恢复精度；
  - 该标准作为后续所有阶段的 baseline 对比基准。

---

### v1.0-b：Critic repair / protected TD3

* **目标**：防止普通 TD3 fine-tune 破坏 BC policy。

* **机制清单**：
  - **replay prefill**：用 BC 数据预填充 replay buffer，确保 critic 在训练初期能接触到高质量、分布正确的数据；
  - **critic warmup**：先独立训练 critic 若干步，让 Q 网络学会基本的价值面，再放开 actor 更新；
  - **actor freeze**：在 critic 充分 warmup 前，冻结 actor 网络，防止 actor 在 critic 尚未学会时盲目更新；
  - **BC regularization**：actor loss 中加入 BC 行为克隆正则项，约束 actor 不偏离 BC policy；
  - **low actor_lr**：降低 actor 学习率，减缓 actor 更新速度；
  - **critic audit**：定期审计 critic 是否正确区分好/坏动作；
  - **rollout-based model selection**：以实际 rollout 表现选模型，而非 episode reward。

---

### v1.0-c：Residual actor 验证

* **网络形式**：`a = clip(π_BC(s) + α Δπθ(s), -1, 1)`

* **核心设计**：
  - π_BC 冻结，不更新；
  - 只训练 Δπθ（residual actor）；
  - 第一版只支持 gimbal_only；
  - residual_scale α 必须保守（例如 0.05 / 0.1 / 0.2）。

* **验收要求**：
  - **不劣于 BC**，而不是只看 episode reward；
  - 固定角度 rollout eval 对比 BC baseline；
  - `action_abs_mean` 保持小动作区；
  - `sat_rate` 接近 0。

---

**5. Critic calibration gate（v1.0 新增）**

如果 critic audit 显示以下任一情况，则不允许放开 actor 更新，只继续 critic warmup / replay prefill / recovery data：

- `Q(s, bad action) > Q(s, BC action)` — critic 给坏动作的 Q 值高于 BC 动作；
- `Q(s, sat+) / Q(s, sat-) > Q(s, BC action)` — critic 给饱和动作的 Q 值高于 BC 动作；
- bad states 上 `frac_action_gt_BC` 过高 — 在偏离状态下，critic 将非 BC action 评为优于 BC action 的比例过高；
- Q 面过平、margin 不足 — critic 无法有效区分不同动作的价值。

**Critic calibration gate 是 actor 更新的硬性前置条件。**

---

**6. Recovery data（v1.0 新增）**

* **replay prefill 不应只有成功 BC 轨迹**；
* 还应包含偏离状态下的 recovery action；
* 目标是让 critic 学会 bad states 下如何回到安全区域；
* 可通过以下方式生成 recovery data：
  - 在 BC rollout 中注入小幅扰动，记录 recovery 轨迹；
  - 使用 DAgger-style 数据闭环：让 actor 探索，BC 在偏离状态下接管并记录 recovery action；
  - 在 replay buffer 中保持 recovery data 的固定比例。

---

**7. 动力学与动作流转 (env.step)**

* **动作接口分层**：
  - **full_8d（8 维）**是最终动作接口，前 4 维为框架指令、后 4 维为飞轮指令；
  - **gimbal_only（4 维）**是当前 debug 和 v1.0 前置训练使用的中间动作模式，只输出框架指令，飞轮转速维持 3000 rpm 固定偏置；
  - gimbal_only 未稳定前，不直接推进 full_8d；
  - 当前重点不是修改 VSCMG 几何或物理参数，而是修复 TD3 训练机制（普通 fine-tune 导致策略退化）。

* **双态空间解耦 (Dual-State Mapping)**：底层运动学积分强制使用四元数 (Quaternion)，利用其全域无奇异的特性保证计算鲁棒性。按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再通过公式 `σ_err = q_err_vec / (1 + q_err0)` 将其转换为 3 维修正罗德里格参数 (MRP)，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`），将 `σ_err` 作为姿态观测值填入 22 维 State 向量中。误差四元数定义采用项目统一约定，环境、奖励、测试、可视化全部遵循同一约定，该约定在实现说明或数学附录中单点记录。

* **非对称动作缩放**：网络输出的 [-1, 1] 必须被精确解析。前 4 维（框架指令）最大缩放至 1 rad/s；后 4 维（飞轮指令）最大缩放至 50 rad/s^2。

* **Reward 基于姿态误差**：Reward 计算基于 `sigma_err`（3维）姿态误差，符合控制理论的反馈原则。



**8. 验收标准**

* **评估方式**：使用 `env.reset(options={"init_attitude_deg": angle})` 进行固定角度 rollout 评估。

* **gimbal_only 模式收敛要求**：
  - 10° / 20° / 30° 在 `max_steps=1000` 下稳定收敛；
  - 45° 有明显改善。

* **动作质量指标**：
  - `action_abs_mean` 保持小动作区（不出现高饱和大动作）；
  - `sat_rate` 接近 0。

* **选模标准**：
  - 不再只用 `best_episode_reward.pth` 选模；
  - 必须通过固定角度 rollout eval 验证实际控制表现；
  - rollout-based model selection 已替代 best_episode_reward。

* **策略质量下限**：
  - protected TD3 训练得到的策略不劣于 BC actor baseline；
  - residual actor 不劣于 BC baseline；
  - full_8d 扩展前需要单独评估，不假设 gimbal_only 的成功可直接迁移。

* **可复现性**：训练结果需要可复现，需记录 seed、config 和 eval 脚本。

* 飞轮转速基本维持在 3000 rpm 附近，不出现无意义饱和。

---

**9. 当前 debug 结论（v1.0 阶段）**

- **BC actor 仍是全局最稳 baseline**；
- **bc_reg_weight=1.0 / 10k 是当前最优 protected TD3 candidate**；
- **bc_reg_weight=1.0 / 20k 已出现相对 BC 的过程质量退化**；
- **所以不继续盲目加长普通 TD3 / BC-reg 训练，而转向 critic repair + residual actor**。
