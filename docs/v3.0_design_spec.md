# 架构设计规约：v3.0 抗扰压榨期 — Robust Residual Sim-to-Real Preparation

**前置依赖 / 当前状态说明**

> v3.0 是 v2.0 稳定后的远期目标，不是当前工作重点。在训练机制稳定之前，不应启动 v3.0。

v3.0 进入前必须满足：

1. **v2.0 full_8d 稳定控制器已就位**：v2.0 全域机动 residual actor 已通过验收，critic ensemble / uncertainty gate 有效。
2. **residual actor 训练机制已充分验证**：惯量随机化、扰动、延迟会显著增加 critic 学习难度，必须等 residual actor + critic calibration 在 v2.0 下充分验证后再启用。
3. **critic ensemble / uncertainty gate 已作为核心机制**：v3.0 的 domain randomization 需要 ensemble 来处理分布外的价值估计。
4. **safety shield 已在 v2.0 中验证**：gimbal / wheel safety 机制已具备，v3.0 在此基础上增加扰动环境下的安全约束。
5. **参数接口不等于充分条件**：当前 env_config / dynamics 侧已具备惯量偏差、伺服延迟、扰动力矩的参数接口，但这只是必要条件，不是训练稳定性的充分条件。

**风险说明**：过早引入扰动和延迟会导致 critic 难以学到准确的价值函数，进而拖垮 actor 更新。如果基础训练流程本身不稳定，叠加域随机化只会放大问题而非解决它。

---

**1. 核心目标**

对抗真实航天器的硬件缺陷与太空恶劣环境干扰，打造工业级鲁棒性，确保算法具备硬件在环 (HIL) 部署能力。v3.0 定位为 **Robust residual sim-to-real preparation**。

**网络形式**：
```
a = robust_base(s) + α Δπθ(s)
```
其中：
- robust_base(s)：v2.0 得到的稳定 baseline；
- Δπθ(s)：学习模型误差、延迟、扰动补偿的 residual actor。

---

**2. 环境初始化 (env.reset)**

* **域随机化 (Domain Randomization)**：卫星转动惯量矩阵 J 必须在标称值基础上随机波动 ±15%。

---

**3. Domain randomization 不只是随机撒参数（v3.0 强化）**

v3.0 的 domain randomization 应遵循以下原则：

* **System identification + domain randomization**：
  - 先通过系统辨识获取真实参数范围；
  - 随机化范围应基于真实不确定性，而非任意设定；
  - 参数敏感性分析：在随机化之前，先评估每个参数对控制性能的敏感度。

* **随机范围过大的风险**：
  - 随机范围过大可能导致策略过度保守；
  - 应逐步扩大随机化范围，而非一步到位；
  - 每次扩大范围后，必须通过 critic audit / rollout eval 验证。

---

**4. 动力学与动作流转 (env.step)**

* **状态语义继承**：v3.0 完全保持 v1.0 的 22 维状态语义定义，不做任何修改。

* **双态空间解耦 (Dual-State Mapping)**：底层运动学积分强制使用四元数 (Quaternion)，利用其全域无奇异的特性保证计算鲁棒性；积分完成后，按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再通过公式 `σ_err = q_err_vec / (1 + q_err0)` 将其转换为 3 维修正罗德里格参数 (MRP)，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`），将 `σ_err` 作为姿态观测值填入 22 维 State 向量中。

* **伺服延迟注入**：在动力学方程前，给电机指令加入一阶系统滞后。

* **外扰力矩注入**：加入周期性及白噪声形式的空间扰动力矩。

---

**5. Critic 机制（v3.0 强化）**

* **非对称 Actor-Critic 保留**：训练期给予 Critic 网络"上帝视角"，Actor 仅获取带噪声观测值，稳定恶劣环境下的梯度。

* **Critic ensemble 从 v3.0 起作为核心机制**：
  - ensemble member 数量建议 3~5 个；
  - 用于 uncertainty estimation 和 pessimistic Q 估计；
  - actor 更新受 uncertainty gate 约束。

* **Critic privileged information**：
  - 精确惯量参数；
  - 延迟参数；
  - 扰动力矩参数或估计量；若实现支持，则可包含近期历史；
  - 无噪声状态。

* **Actor 仍只使用可测 22D observation**，不使用任何 privileged information。

---

**6. 验收标准**

* 在惯量偏差与伺服延迟下，稳态不出现高频极限环震荡。

* 遭遇突发环境扰动时，能迅速重新咬住目标姿态。

* **Critic ranking / uncertainty audit 通过**：
  - ensemble 内 critic 排名稳定；
  - uncertainty 在 OOD 状态下显著升高；
  - uncertainty 在 in-distribution 状态下保持低位。

* **Domain randomized rollout matrix 通过**：
  - 在多种随机化参数组合下，rollout 表现不低于 v2.0 robust_base；
  - 不出现因随机化参数组合导致的catastrophic failure。

* **不出现极限环**：在各种域随机化参数下，residual actor 不引入高频震荡。

* **不因 residual actor 进入 bad state distribution**：
  - residual actor 的输出不导致系统偏离 baseline 可恢复范围；
  - safety shield 不应频繁触发。

* **Safety shield 不应频繁触发**：
  - 频繁触发表明 residual actor 在探索危险区域；
  - 应降低 α 或增加 critic warmup。
