# 架构设计规约：v3.0 抗扰压榨期

**前置依赖 / 当前状态说明**

> v3.0 是 v2.0（至少 v1.x/full_8d）稳定后的远期目标，不是当前工作重点。在训练机制稳定之前，不应启动 v3.0。

v3.0 进入前必须满足：

1. **稳定可训练控制器已就位**：至少完成 v1.x/full_8d 稳定控制器，理想情况下完成 v2.0 后再进入 v3.0。
2. **TD3 训练机制已修复**：在普通 TD3 fine-tune 问题解决前，不应引入 domain randomization、伺服延迟、扰动力矩。这些因素会掩盖训练不稳定问题，增加排查难度。
3. **protected TD3 / BC-assisted TD3 训练流程已稳定**：惯量随机化、扰动、延迟会显著增加 critic 学习难度，必须等保护机制充分验证后再启用。
4. **参数接口不等于充分条件**：当前 env_config / dynamics 侧已具备惯量偏差、伺服延迟、扰动力矩的参数接口，但这只是必要条件，不是训练稳定性的充分条件。
5. **非对称 Actor-Critic 保留为长期方向**：不作为当前下一步，等 v1.0/v2.0 基础训练流程稳定后再评估引入时机。

**风险说明**：过早引入扰动和延迟会导致 critic 难以学到准确的价值函数，进而拖垮 actor 更新。如果基础训练流程本身不稳定，叠加域随机化只会放大问题而非解决它。

---

**1. 核心目标**

对抗真实航天器的硬件缺陷与太空恶劣环境干扰，打造工业级鲁棒性，确保算法具备硬件在环 (HIL) 部署能力。



**2. 环境初始化 (env.reset)**

* **域随机化 (Domain Randomization)**：卫星转动惯量矩阵 J 必须在标称值基础上随机波动 ±15%。



**3. 动力学与动作流转 (env.step)**

* **状态语义继承**：v3.0 完全保持 v1.0 的 22 维状态语义定义，不做任何修改。

* **双态空间解耦 (Dual-State Mapping)**：底层运动学积分强制使用四元数 (Quaternion)，利用其全域无奇异的特性保证计算鲁棒性；积分完成后，按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再通过公式 `σ_err = q_err_vec / (1 + q_err0)` 将其转换为 3 维修正罗德里格参数 (MRP)，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`），将 `σ_err` 作为姿态观测值填入 22 维 State 向量中。

* **伺服延迟注入**：在动力学方程前，给电机指令加入一阶系统滞后。

* **外扰力矩注入**：加入周期性及白噪声形式的空间扰动力矩。

* **非对称 Actor-Critic**：训练期给予 Critic 网络"上帝视角"（输入无噪声状态与精确惯量），Actor 仅获取带噪声观测值，稳定恶劣环境下的梯度。



**4. 验收标准**

* 在惯量偏差与伺服延迟下，稳态不出现高频极限环震荡。

* 遭遇突发环境扰动时，能迅速重新咬住目标姿态。
