# 架构设计规约：v4.0 学术巅峰期 — Token/Attention Residual Actor

**当前状态说明 / 前置依赖**

> v4.0 是长期学术目标，不作为当前工作重点。在 v3.0 robust residual controller 稳定前，不优先切换网络架构。

1. **当前 MLP 表达能力已足够**：BC actor 使用现有 MLP actor 结构已经能表达有效的姿态控制策略。
2. **瓶颈是训练机制，不是网络架构**：普通 TD3 fine-tune 会破坏 BC policy，问题出在 critic/actor 更新机制，而非 MLP 表达能力不足。
3. **Token / Attention 架构保留为长期学术目标**：在 protected TD3 训练流程稳定前，换网络不能替代 residual actor、critic calibration、safety gate 等基础机制。
4. **v4.0 依赖前序阶段稳定**：只有当 v1.0~v3.0 的训练流程和控制器都稳定后，才有基础评估是否需要更强的网络架构。

**风险说明**：过早切换到 Token/Attention 架构会增加调试复杂度，且无法判断性能改善来自架构升级还是训练机制修复。必须在训练机制稳定后再评估架构升级的收益。

---

**1. 核心目标**

重构神经网络前端，提升多执行机构协同的天花板，解决深度强化学习在航天控制领域的"黑盒"不可解释难题。

v4.0 不把策略重新变回普通 direct actor。v4.0 的定位是 **Token/Attention residual actor**：

```
tokens = tokenize_22d_state(s)
Δπθ = AttentionResidualActor(tokens)
a = base(s) + α Δπθ
```

Attention 是表达能力升级，不替代 residual actor / critic calibration / safety gate 主线。

---

**2. 网络架构改造**

* **v4.0 不重新定义状态空间**：v4.0 将**同一套冻结的 22 维物理语义**重新打包为实体 Token，不是重新定义状态空间。

* **推荐的 Token 打包方案**：
  - **本体 Token**：`[sigma_err（3维）, omega_B（3维）]` — 6维，航天器本体的姿态误差与角速度状态
  - **VSCMG Token**（4个）：每个为 `[sin(delta_i), cos(delta_i), delta_dot_i, Omega_tilde_i]` — 4维，第 i 个 VSCMG 的完整执行器状态
  - **总计**：1 个本体 Token（6维）+ 4 个 VSCMG Token（各4维）= 仍为 22 维

* **物理引导注意力 (Self-Attention)**：引入注意力机制，将 VSCMG 的物理安装几何矩阵（倾角与方位角）作为位置编码 (Positional Encoding) 注入网络。

* **知识迁移策略**：v4.0 在保持 22 维状态语义不变的前提下，尽量复用既有策略知识。是否可直接加载旧 checkpoint，由具体实现决定；必要时采用部分权重迁移、adapter 或微调策略。

---

**3. 动力学约束**

* **状态语义继承**：v4.0 完全保持 v1.0 的 22 维状态语义定义，仅改变前端编码方式。

* **双态空间解耦 (Dual-State Mapping)**：底层运动学积分强制使用四元数 (Quaternion)，利用其全域无奇异的特性保证计算鲁棒性；积分完成后，按照项目统一采用的姿态误差约定形成误差四元数 `q_err`，对 `q_err` 做双覆盖规避（若 `q_err0 < 0` 则整体取反），再通过公式 `σ_err = q_err_vec / (1 + q_err0)` 将其转换为 3 维修正罗德里格参数 (MRP)，最后做影子集切换（若 `||σ_err||² > 1` 则 `σ_err = -σ_err / ||σ_err||²`），将 `σ_err` 作为姿态观测值填入 22 维 State 向量中。

* 继承 v3.0 的全量地狱级难度（延迟、扰动、惯量偏差）。

---

**4. Critic ensemble 保留**

* Attention 改善表达能力；
* ensemble / uncertainty gate 负责可信度；
* 两者不是替代关系；
* v4.0 的 critic 可继续使用 MLP 或升级为 Attention-based critic，但 ensemble / uncertainty gate 机制必须保留。

---

**5. 可选长期升级：null-space residual actor**

v4.0 可选引入 null-space residual actor：

```
a = a_base(s) + α N(s) zθ(tokens)
```

其中：
- N(s)：null-space projector，基于 VSCMG control allocation matrix B(s)；
- zθ(tokens)：null-space 方向的 residual，理论上不影响任务空间输出。

**前置条件（必须全部满足才可引入）**：
- VSCMG control allocation matrix B(s) 已审计，精度满足要求；
- null-space projector N(s) 已验证，在参数辨识后的真实近似参数下仍有效；
- 不准确的 N(s) 会把"零运动"变成真实扰动；
- 所以 null-space residual 不应在 v1~v3 过早引入。

---

**6. 验收标准**

* 作为长期学术目标，极端扰动下的最终指向精度力争突破至 10^-4 量级。

* 可通过导出 Attention 权重热力图，直观解释 AI 分配 8 个电机的发力几何逻辑。

* residual actor 不劣于 v3.0 baseline（robust_base(s)）。

* critic ensemble / uncertainty gate 在 Attention 架构下仍有效。

* safety shield 不频繁触发。
